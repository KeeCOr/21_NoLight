# 액션 플랫포머 설계 문서

**날짜**: 2026-05-14
**프로젝트**: 21_NL (가칭)
**플랫폼**: Windows (Electron + Phaser.js)

---

## 1. 게임 개요

동양 수묵화 배경에 메카닉 요소가 혼합된 캐릭터가 등장하는 2D 액션 플랫포머.
플레이어는 2명(이후 확장 가능)의 캐릭터를 실시간으로 교체하며 전투하고, 강력한 추격자를 피해 무한 생성 맵을 이동한다.
명확한 엔딩 없이 최대한 오래 생존하며 점수를 쌓는 방식이다.

---

## 2. 핵심 메커니즘

### 2-1. 저체력 강화 (Risk & Reward)

- 적을 처치하지 않으면 HP가 지속적으로 감소한다
- HP가 낮을수록 공격력/스킬 위력이 증가한다
- HP가 낮을수록 스태미나 자동 충전 속도가 빨라진다
- HP는 두 캐릭터가 공유한다

### 2-2. 캐릭터 교체

- `Tab` 또는 `Shift`로 즉시 교체
- 교체 순간 `onSwapIn()` 특수 효과 발동
- HP 공유, 스태미나 공유
- 캐릭터 풀 구조로 3번째 캐릭터 추가 시 코드 변경 최소화

### 2-3. 스태미나

- 시간당 자동 충전 (저체력 시 충전 속도 가속)
- 메카 팔 캐릭터 가드 피격 시 추가 충전
- 가드 유지 중 소모
- 스킬 발동 시 소모

## 3. 전체 아키텍처

```
Electron
└── BrowserWindow
    └── Phaser.js Game
        ├── Scenes
        │   ├── BootScene        — 에셋 로딩
        │   ├── MainMenuScene    — 타이틀 화면
        │   ├── GameScene        — 메인 게임플레이
        │   └── GameOverScene    — 점수 표시
        ├── Systems
        │   ├── MapGenerator     — 무한 청크 생성/제거
        │   ├── CharacterManager — 캐릭터 교체 및 스왑 이펙트
        │   ├── PursuerAI        — 추격자 행동 및 피격 처리
        │   └── StatSystem       — HP, 스태미나, 점수 관리
        └── Entities
            ├── Player           — 현재 조작 캐릭터
            ├── Characters[]     — 교체 가능한 캐릭터 풀
            ├── Pursuer          — 추격자
            └── Enemies[]        — 일반 적
```

**데이터 흐름**: GameScene이 매 프레임 Systems를 업데이트 → Systems가 Entities 상태를 변경 → Phaser 렌더러가 출력

## 4. 맵 생성 시스템

- 맵은 **청크(Chunk)** 단위로 관리 (화면 너비 x 약 300px 높이)
- 플레이어가 위/아래로 이동하면 새 청크를 생성하고, 화면 밖으로 멀어진 청크는 제거
- 발판 밀도, 간격, 크기를 랜덤 시드로 결정
- 최소 이동 가능 경로는 항상 보장 (막힌 맵 방지)
- 적은 청크 생성 시 함께 스폰, 일정 밀도 유지
- 추격자는 항상 플레이어 아래(또는 위) 고정 오프셋에 위치

## 5. 캐릭터 시스템

### 공통 베이스

```
BaseCharacter
├── stats: { hp(공유), atk, speed, stamina(공유) }
├── attack()        — 기본 공격
├── skill()         — 고유 스킬 (스태미나 소모)
├── guard()         — 가드 (캐릭터별 효과, 스태미나 소모)
├── parry()         — 피격 직전 가드 시 공통 발동
│                       → 투사체 반사 + 적 공격 캔슬
└── onSwapIn()      — 교체 진입 시 특수 효과
```

### 전기 범위형 캐릭터

```
ElectricCharacter
├── attack(): 전방 범위 전기 방출
├── skill(): 광역 전기 폭발 (차지 후 발동)
├── guard(): 역장 전개 → 주변 투사체 슬로우
└── onSwapIn(): 주변 적 즉시 감전 효과
```

### 메카 팔 근접형 캐릭터

```
MechaArmCharacter
├── attack(): 빠른 3단 콤보
├── skill(): 메카 팔 대시 돌진
├── guard(): 대미지는 받되 스태미나 추가 충전
└── onSwapIn(): 짧은 무적 + 속도 부스트
```

### 패링

- 가드 입력 후 약 6~10프레임(100ms 내외, 구현 시 조정) 이내 피격 시 패링 판정
- 투사체 반사, 적 공격 캔슬
- 두 캐릭터 공통 적용

---

## 6. 추격자 시스템

- 항상 플레이어 방향으로 일정 속도로 이동
- 접촉 시 지속 대미지 (즉사 아님)
- 복합 공격 패턴 (순환 또는 랜덤)
  - **투사체 발사**: 주기적으로 플레이어 방향 조준
  - **범위 충격파**: 예고 이펙트 후 주변 영역 폭발
  - **돌진**: 가끔 빠르게 플레이어 방향으로 돌격
- **저지 메커니즘**: 누적 대미지가 임계값에 도달하면 일시 정지 (3~5초)
  - 정지 후 재개, 시간이 지날수록 임계값 증가 (저지 점점 어려워짐, 증가 속도는 밸런싱 단계에서 확정)
- **난이도 곡선**: 생존 시간에 따라 이동 속도/공격 빈도 증가
- 패링으로 투사체 반사 및 돌진 캔슬 가능

---

## 7. 스탯 시스템

```
StatSystem
├── HP (공유)
│   ├── 적 미처치 시 지속 감소
│   ├── 낮을수록 공격력/스킬 강화
│   └── 낮을수록 스태미나 충전 속도 증가
└── 스태미나
    ├── 시간당 자동 충전 (저체력 시 가속)
    ├── 메카 팔 가드 피격 시 추가 충전
    ├── 가드 유지 중 소모
    └── 스킬 발동 시 소모

점수 = 생존 시간 + 처치 적 수 기반
```

## 8. 비주얼 & 오디오

### 수묵화 스타일

- **배경**: 먹물 번짐 느낌의 레이어드 배경 (원경/근경 패럴랙스)
- **캐릭터/적**: 수묵화 풍 스프라이트 + 메카 부품은 금속 질감 대비
- **이펙트**: 전기 공격은 먹물이 튀는 느낌, 메카 팔은 충격파 선화
- **UI**: 붓글씨 느낌의 폰트, 한지 질감 HUD

### HUD 구성

- HP 바 (좌측 상단)
- 스태미나 바 (HP 바 하단)
- 현재/대기 캐릭터 아이콘 (우측 하단)
- 생존 시간 + 점수 (상단 중앙)

### 오디오

- BGM: 전통 타악기 + 전자음 혼합
- SFX: 공격/피격/교체 각각 구분되는 효과음
