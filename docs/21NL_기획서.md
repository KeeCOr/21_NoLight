# 묵철 (墨鐵) - 게임 기획서

**프로젝트명**: 21_NL
**정식 명칭**: 묵철 (墨鐵)
**장르**: 수묵화 스타일 2D 액션 플랫포머
**플랫폼**: Windows (Electron + Phaser.js)
**현재 버전**: v0.1.0
**최종 수정일**: 2026-05-15

---

## 1. 게임 개요

동양 수묵화 배경에 메카닉 요소가 혼합된 캐릭터가 등장하는 2D 액션 플랫포머.
플레이어는 2명의 캐릭터를 실시간으로 교체하며 전투하고, 강력한 추격자를 피해 무한 생성 맵을 이동한다.
명확한 엔딩 없이 최대한 오래 생존하며 점수를 쌓는 서바이벌 방식이다.

### 게임 목표
- 적을 처치하고 추격자를 저지하며 최대한 오래 생존
- 점수 = 생존 시간(초) x 10 + 처치 수 x 50

---

## 2. 조작법

| 키 | 동작 |
|----|------|
| 방향키 좌/우 | 이동 (속도 220) |
| 방향키 위 | 점프 (지면 접촉 시만, 속도 -550) |
| Z | 기본 공격 |
| X | 스킬 (스태미나 30 소모) |
| C (홀드) | 가드 (스태미나 초당 15 소모) |
| Tab | 캐릭터 교체 |
| Space | 게임 시작 / 재시작 |
| ESC | 메인 메뉴로 복귀 (게임오버 화면에서) |

## 3. 핵심 메커니즘

### 3-1. 저체력 강화 (Risk & Reward)

적을 처치하지 않으면 HP가 지속적으로 감소한다. HP가 낮을수록 강해지는 시스템.

| 조건 | 효과 |
|------|------|
| HP 30 이하 (저체력) | 공격력 1.5배 |
| HP 30 이하 (저체력) | 스태미나 자동 충전 2배 (초당 10 -> 20) |
| 적 미처치 3초 이상 경과 | HP 초당 2씩 감소 |
| 적 처치 시 | HP 감소 3초간 유예 |

### 3-2. 캐릭터 교체

- Tab 키로 즉시 교체
- 교체 순간 새 캐릭터의 `onSwapIn()` 특수 효과 발동
- HP와 스태미나는 두 캐릭터가 공유
- 교체된 캐릭터는 이전 캐릭터의 위치에 등장
- 캐릭터 풀 구조로 3번째 캐릭터 추가 시 코드 변경 최소화

### 3-3. 스태미나

| 상황 | 수치 |
|------|------|
| 최대치 | 100 |
| 자동 충전 (일반) | 초당 10 |
| 자동 충전 (저체력) | 초당 20 |
| 가드 유지 중 소모 | 초당 15 |
| 스킬 발동 소모 | 30 |
| 메카 팔 가드 피격 시 획득 | 20 |

### 3-4. 패링

- 가드(C키) 입력 후 100ms 이내 피격 시 패링 판정
- 패링 성공 시: 투사체 반사 + 적 공격 캔슬
- 두 캐릭터 공통 적용

### 3-5. 무적

- 피격 시 500ms 무적 (반투명 표시)
- 캐릭터별 특수 무적 시간 별도 존재

## 4. 캐릭터 시스템

### 4-1. 전기 범위형 캐릭터 (ElectricCharacter)

범위 공격과 역장으로 다수의 적을 제어하는 캐릭터.

| 행동 | 설명 | 수치 |
|------|------|------|
| 기본 공격 (Z) | 전방 범위 전기 방출 + 스턴 | 대미지 20, 사거리 200, 스턴 500ms |
| 스킬 (X) | 광역 전기 폭발 + 스턴 | 대미지 50, 반경 300, 스턴 1000ms, 스태미나 30 소모 |
| 가드 (C) | 역장 전개, 투사체 속도 30%로 감속 | 반경 250, 피격 시 대미지 50% 경감 |
| 교체 진입 | 주변 적 즉시 감전 스턴 | 반경 200, 스턴 800ms |

### 4-2. 메카 팔 근접형 캐릭터 (MechaArmCharacter)

빠른 콤보와 대시로 근접전에 특화된 캐릭터.

| 행동 | 설명 | 수치 |
|------|------|------|
| 기본 공격 (Z) | 3단 콤보 (600ms 내 연속 입력) | 기본 대미지 15, 콤보 배율 1.0/1.2/1.5, 사거리 80/100/120 |
| 스킬 (X) | 메카 팔 대시 돌진 + 무적 | 속도 800, 지속 200ms, 무적 200ms, 스태미나 30 소모 |
| 가드 (C) | 풀 대미지를 받되 스태미나 충전 | 피격 시 대미지 100% + 스태미나 20 획득 |
| 교체 진입 | 짧은 무적 + 속도 부스트 | 무적 800ms, 속도 1.5배 (1초간) |

## 5. 적 시스템

### 5-1. 일반 적 (Enemy)

청크 생성 시 함께 스폰되는 기본 적.

| 항목 | 수치 |
|------|------|
| HP | 50 |
| 이동 속도 | 80 |
| 공격 대미지 | 10 |
| 공격 사거리 | 60 |
| 공격 쿨다운 | 1500ms |
| 청크당 스폰 수 | 1~3마리 |

- 플레이어 방향으로 접근하여 근접 공격
- 스턴 적용 가능 (전기 캐릭터 공격 등)
- 처치 시 HP 감소 유예 및 점수 획득

### 5-2. 추격자 (Pursuer)

게임 내내 플레이어를 추적하는 강력한 보스급 존재. 파괴 불가.

| 항목 | 수치 |
|------|------|
| 기본 이동 속도 | 120 |
| 접촉 대미지 | 5 (0.5초마다) |
| 접촉 판정 거리 | 60 |
| 중력 | 없음 (공중 이동) |

#### 추격자 공격 패턴 (순환)

| 순서 | 패턴 | 간격 | 대미지 |
|------|------|------|--------|
| 1 | 투사체 발사 (플레이어 조준) | 3초 | 15 (속도 300) |
| 2 | 범위 충격파 (예고 0.8초 후 폭발) | 6초 | 20 (반경 200) |
| 3 | 돌진 (플레이어 방향) | 8초 | 접촉 대미지 (속도 500, 0.4초) |

#### 저지 메커니즘

- 누적 대미지가 임계값에 도달하면 4초간 정지 (스턴)
- 초기 임계값: 200
- 저지 성공마다 임계값 +50 증가 (점점 저지 어려워짐)
- 패링으로 투사체를 반사하면 추격자에 30 대미지

#### 난이도 곡선

- 30초마다 이동 속도 5% 증가
- 공격 패턴 간격은 고정 (향후 밸런싱에서 조정 가능)

## 6. 맵 시스템

### 청크 기반 무한 생성

- 맵은 청크(Chunk) 단위로 관리 (높이 300px)
- 플레이어 위치 기준 상하 2개 청크 범위를 유지
- 범위 밖 청크는 자동 제거 (메모리 관리)
- 최초 청크(인덱스 0)에는 적이 스폰되지 않음

### 발판 배치

| 항목 | 수치 |
|------|------|
| 보장 발판 | 좌(x:200)/중(x:640)/우(x:1080) 각 1개 (너비 120) |
| 추가 발판 | 2~4개 (랜덤 위치, 너비 60~150) |
| 발판 높이 | 20px |

- 좌/중/우 3개의 보장 발판으로 이동 경로 항상 확보
- 추가 발판으로 다양한 루트 제공

## 7. HUD (화면 표시)

| 요소 | 위치 | 설명 |
|------|------|------|
| HP 바 | 좌측 상단 | 빨간색, HP 30% 이하 시 주황색으로 변경 |
| 스태미나 바 | HP 바 하단 | 파란색 |
| 생존 시간 + 점수 | 상단 중앙 | "MM:SS  N점" 형식 |
| 캐릭터 아이콘 | 우측 하단 | 활성 캐릭터 흰색, 비활성 회색 |

## 8. 게임 흐름

```
Boot (에셋 로딩)
  |
  v
MainMenu ("묵철 (墨鐵)" 타이틀, SPACE로 시작)
  |
  v
GameScene (메인 게임플레이)
  |  - 캐릭터 조작, 적 처치, 추격자 회피
  |  - HP가 0이 되면 게임오버
  v
GameOver (점수/시간 표시, SPACE 재시작 / ESC 메뉴)
```

## 9. 기술 스택

| 구분 | 기술 | 버전 |
|------|------|------|
| 런타임 | Electron | 28.x |
| 게임 엔진 | Phaser.js | 3.60 |
| 언어 | JavaScript | ES6+ |
| 테스트 | Jest | 29.x |
| 빌드 | electron-builder | 24.x |
| 배포 | Windows Portable EXE | - |

### 아키텍처

```
Electron
 +-- BrowserWindow
      +-- Phaser.js Game
           +-- Scenes
           |    +-- BootScene        (플레이스홀더 에셋 생성)
           |    +-- MainMenuScene    (타이틀 화면)
           |    +-- GameScene        (메인 게임플레이)
           |    +-- GameOverScene    (점수 표시)
           +-- Systems
           |    +-- StatSystem       (HP/스태미나/점수 - 순수 로직)
           |    +-- CharacterManager (캐릭터 교체 - 순수 로직)
           |    +-- MapGenerator     (무한 청크 생성/제거)
           |    +-- PursuerAI        (추격자 상태머신 - 순수 로직)
           +-- Entities
           |    +-- BaseCharacter    (캐릭터 공통 베이스)
           |    +-- ElectricCharacter(전기 범위형)
           |    +-- MechaArmCharacter(메카 팔 근접형)
           |    +-- Enemy            (일반 적)
           |    +-- Pursuer          (추격자)
           +-- UI
                +-- HUD              (게임 내 정보 표시)
```
