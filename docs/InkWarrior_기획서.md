# InkWarrior 기획서

> 버전 v0.3 · 2026-06-25

---

## 문제 정의

**어떤 유저가:** 짧은 틈새 시간(5~20분)에 즉각적인 타격감과 시각 피드백을 즐기고 싶은 PC/인디 게임 유저  
**어떤 상황에서:** 전투 조작이 맞았는지, 회피가 성공했는지, 얼마나 피해를 입었는지가 화면에서 즉시 읽히지 않아 "지금 내가 잘 싸우고 있는지" 판단하기 어려울 때  
**어떤 게임적 욕구를:** 공격을 넣는 순간 카메라 흔들림·먹 번짐·콜아웃이 동시에 반응하면서 "내가 제대로 싸우고 있다"는 확신과 쾌감을 얻고 싶다 — 수묵화 비주얼이 단순한 장식이 아니라 전투 정보 전달의 핵심 언어가 되는 경험을 원한다.

## 주 페르소나

**이름:** 이도현 (26세, 대학원생·인디게임 팬)  
**직업:** 디자인 전공 대학원생, 부업으로 일러스트 프리랜서  
**현재 게임 습관:** 수업·작업 사이 20분 내외 쉬는 시간에 스팀 인디게임을 즐긴다. Dead Cells, Hollow Knight처럼 타격감이 명확하고 시각 언어가 통일된 게임을 선호하며, 한 번 시작하면 "딱 한 판만 더"가 자연스럽게 나오는 루프를 좋아한다.  
**이 게임을 플레이하는 맥락:** 수묵화 비주얼에 매력을 느끼지만 예쁜 그래픽만으로는 오래 안 한다. 공격을 넣었을 때 카메라 흔들림·먹 번짐·콜아웃이 동시에 반응하는 순간, "내가 제대로 싸우고 있다"는 확신과 쾌감을 느낀다.  
**기대하는 경험:** 5분 짜리 세션에서도 "이번엔 콤보를 더 이어봤어야 했는데" 하고 아쉬워하며 즉시 재시작하게 만드는 구조. 캐릭터 교체 타이밍이 내 실력처럼 느껴질 것.

---

## 핵심 루프

```
전장 상황과 추격자 거리를 확인한다
  → 공격, 회피, 스킬, 캐릭터 교체 중 현재 행동을 선택한다
  → 타격/회피/피해/처치 결과가 bitmap VFX와 콜아웃으로 즉시 표시된다
  → 점수, 회복 아이템, HP와 스태미나 변화를 확인한다
  → 다음 청크와 다음 전투로 이동한다
```

---

## MVP 가설

1. **[strike/evade/wound/finish 통합 피드백 언어]가 있으면** 유저가 복잡한 전투 상황에서도 "지금 뭐가 일어났는지"를 0.5초 안에 판독해 몰입이 끊기지 않고 세션이 자연스럽게 연장될 것이다.
2. **[카메라 타격·먹 번짐·콜아웃의 동시 발동]이 있으면** 유저가 콤보 성공 순간에 강한 보상감을 느끼고 "다시 그 느낌을 내보고 싶다"는 재플레이 의욕이 생길 것이다.
3. **[전기형·메카 암 캐릭터 교체 시스템]이 있으면** 유저가 적 종류와 상황에 따라 교체 타이밍을 스스로 판단하게 되어 숙련감이 쌓이고 숙련 보상을 더 깊이 체감할 것이다.
4. **[추격자 압박이 시각(비네팅)·수치(거리 바) 두 채널로 동시 표시]되면** 유저가 위기 상황을 직관적으로 인식해 탈출/교전 결정을 빠르게 내리고 긴장감이 지속될 것이다.
5. **[처치 후 회복 드롭과 먹 물줄기 유도선]이 있으면** 유저가 "처치→아이템 획득"의 인과를 시각적으로 이해해 공격적 플레이가 생존에 직접 연결된다는 만족감을 얻을 것이다.

## 레퍼런스 분석

| 게임 | 핵심 행동까지 UX 단계 수 | 적용 교훈 |
|---|---|---|
| **Dead Cells** (메트로배니아 액션) | 맵 진입 → 적 인식 → 공격/회피 → 피드백 확인: **3단계** | 피격·처치 피드백(히트스탑, 화면 효과, 사운드)이 동일한 타이밍 레이어에서 동기화되어야 "내가 잘 싸웠다"는 확신이 생긴다 — InkWarrior CameraImpactProfile의 흔들림·플래시 타이밍을 VFX와 동일 프레임에 맞추는 데 적용 |
| **Hollow Knight** (플랫폼 액션) | 이동 → 적 인식 → 패리/공격 → 영혼 획득 확인: **4단계** | 성공 행동(패리·처치)에는 과장된 시각 보상, 실패(피격)에는 절제된 피드백으로 밸런스 — InkWarrior의 strike/finish는 과장, wound는 절제하는 피드백 계층화에 적용 |
| **Katana ZERO** (픽셀 액션) | 방 진입 → 적 패턴 읽기 → 슬로우모 회피/처치 → 결과 확인: **3단계** | 짧은 콜아웃(텍스트 레이블)이 화면 중앙에 0.5초만 떠도 행동의 의미를 즉시 전달한다 — InkWarrior 콜아웃 시스템의 위치·노출 시간 기준에 직접 적용 |

**이 프로젝트 적용 교훈 핵심:**  
공격·회피·처치 결과를 단일 피드백 언어(strike/evade/wound/finish)로 통합하고, 상위 콤보와 처치 시 카메라·VFX·콜아웃을 동일 프레임에서 동기화해 "내가 제대로 싸우고 있다"는 확신을 0.5초 안에 전달한다.

## 소개용 핵심 메시지

InkWarrior는 "수묵화가 예쁜 게임"이라기보다 "수묵화 비주얼이 전투 피드백 언어가 되는 게임"이다. 플레이어는 공격·회피·피격·처치 모든 순간에 먹 번짐과 카메라 반응이 즉각 따라오는 것을 통해 "내가 잘 싸우고 있는지"를 화면 한 장면으로 판단한다.

- 공격을 넣으면 붓 흔적 VFX와 카메라 흔들림이 동시에 터진다.
- 콤보가 쌓일수록 먹 번짐이 더 강해지고 처치 순간 화면이 터진다.
- 캐릭터 교체 타이밍을 잘 잡으면 그 자체가 나의 실력 표현이 된다.

## 30초 플레이 흐름

| 순서 | 플레이어 행동 | 화면에서 보여야 하는 것 | 재미 포인트 |
|---|---|---|---|
| 1 | 추격자 거리와 HP·스태미나를 확인한다 | 상단 HUD, 추격자 거리 바(녹→황→적), 카메라 가장자리 비네팅 | 위기 수준이 즉시 보여 탈출·교전 판단이 빠르다 |
| 2 | 적 종류를 읽고 캐릭터 교체 여부를 결정한다 | 현재 캐릭터 아이콘, 적 패턴, 교체 쿨다운 표시 | 교체 타이밍이 곧 나의 전략 판단으로 느껴진다 |
| 3 | 공격·회피·스킬 중 행동을 선택한다 | 붓길 예고 콜아웃, brush slash VFX, 스태미나 소비 표시 | 행동 전 예고 콜아웃이 의도와 결과를 연결해준다 |
| 4 | 전투 결과가 즉시 표시된다 | 명중 시 먹 폭발·카메라 흔들림·hit-stop, 피격 시 먹 번짐 경직 콜아웃 | 성공/실패가 0.5초 안에 화면 전체로 전달된다 |
| 5 | 처치 후 회복 드롭을 획득하고 다음 청크로 이동한다 | 먹 물줄기 유도선, 회복 orb 생성, 청크 전환 연출 | "공격적으로 싸울수록 더 오래 산다"는 인과가 보인다 |

## 게임 구성과 규칙 (GDD 요약)

### 화면 구조

| 구역 | 역할 | 구현/검토 메모 |
|---|---|---|
| 상단 HUD | HP, 스태미나, 점수, 캐릭터 교체 쿨다운 표시 | 전투 중 가장 먼저 읽혀야 하는 정보 |
| 전장 플랫폼 | 적 이동·배치, 플레이어 이동·충돌, 청크 전환 처리 | 청크 경계 전환과 낙하/도약 예외가 핵심 테스트 포인트 |
| 추격자 거리 바 | 추격자와의 거리를 색상 바로 실시간 표시 | 화면 우측 에지에 배치, 비네팅과 연동 |
| 콜아웃 레이어 | strike/evade/wound/finish 텍스트 콜아웃 표시 | 같은 프레임에 복수 발생 시 우선순위 큐로 최상위 1개만 표시 |
| VFX 레이어 | brush slash, impact ink burst, blood ink, smoke wisp 등 bitmap VFX | CameraImpactProfile과 동일 프레임에서 동기화 필수 |

### 캐릭터 시스템

| 구분 | 전기형 캐릭터 | 메카 암 캐릭터 |
|---|---|---|
| 기본 역할 | 빠른 연속 공격, 광역 스킬 | 강력한 단타, 방어형 스킬 |
| 강점 | 다수 소형 적 처리, 이동 속도 | 대형 적·보스 처리, 내구성 |
| 교체 조건 | 적 종류·HP 상황에 따라 즉시 교체 | 쿨다운 완료 시 교체 가능 |
| 피드백 특징 | 전기 잔상 + 번개 VFX | 중격 충격파 + 메탈 히트 사운드 |

### 수묵 전투 피드백 규칙

| 피드백 유형 | 콜아웃 | VFX | 카메라 반응 |
|---|---|---|---|
| 기본 공격 (strike) | 붓길 예고 → 참격 콜아웃 | brush slash + impact brush ring | 약한 흔들림 |
| 콤보 공격 | 연타 콜아웃 (2연 참격 등) | combo brush smear + 먹 폭발 | 중간 흔들림 + 줌인 |
| 회피 (evade) | 잔상 콜아웃 | smoke wisp | 방향 넛지 |
| 피격 (wound) | 먹번짐 경직 콜아웃 | blood ink | 짧은 흔들림 |
| 처치 (finish) | 먹물 폭쇄 콜아웃 | impact ink burst | 강한 흔들림 + 플래시 |
| 추격자 러시 | 위험 경고 콜아웃 | 가장자리 붉은 먹 비네팅 | 느린 줌아웃 |

### 추격자 시스템

| 규칙 | 내용 |
|---|---|
| 압박 방식 | 투사체, 쇼크웨이브, 돌진으로 전투 속도를 압박 |
| 거리 표시 | 화면 우측 색상 바(녹→황→적) + 위험 거리 진입 시 붉은 먹 비네팅 |
| 회피 방법 | 적 처치 후 빠르게 다음 청크로 이동해 거리를 벌린다 |
| 패배 조건 | 추격자에 HP가 0이 되거나 본진(HP 바)이 소진 |

### 🖥️ UX / UI
- **콜아웃 과부하:** 같은 프레임에 3개 이상 콜아웃이 겹칠 경우 가장 높은 우선순위 1개만 표시하고 나머지는 소형 아이콘으로 우측 스택 — 수묵 화면 가독성을 유지하면서 주요 피드백 누락 없이 전달.
- **HP·스태미나 바 가독성:** 현재 수묵 배경에 얇은 바가 묻히는 문제를 방지하기 위해 붓질 텍스처 테두리 + 흰 번짐 후광으로 대체하고, 스태미나 소진 시 바 테두리를 1초간 진동 처리.
- **처치/회복 아이템 연결성:** 아이템 생성 위치에서 플레이어 방향으로 먹 물줄기 유도선을 0.3초간 표시해 "처치→회복 드롭→획득" 인과를 시각화, 놓치는 아이템 수 감소.
- **캐릭터 교체 쿨다운 가시성:** 교체 아이콘에 쿨다운 원형 게이지를 추가하고 준비 완료 시 잉크 번짐 애니메이션으로 강조해 교체 가능 타이밍을 직관적으로 인식하게 한다.

### 💰 수익화 전략
- **세션 후 스킨 해금:** 생존 시간·최고 콤보 기준으로 플레이 후 수묵화 스타일의 칼 스킨, 잉크 색상 팔레트(세피아, 청묵, 홍묵) 해금 — 반복 플레이 동기를 외부 결제가 아닌 숙련 보상 구조로 설계해 무료 유저 유지율 선확보 후 유료 DLC 확장.
- **유료 캐릭터 DLC:** 세 번째 캐릭터 이후부터 "필체 유파" 테마 캐릭터(해서체형, 초서체형 등)를 1~2달러 단건 DLC로 판매 — 수묵 테마에 부합하는 네러티브가 있어 이질감 없이 컬렉션 욕구 자극.
- **리더보드 시즌 패스:** 월간 시즌 리더보드에 상위 랭커 전용 수묵 프레임 보상을 추가하고, 시즌 패스(3달러) 구매자에게 중간 보상 2단계 추가 제공 — 경쟁 구조로 DAU 유지.

### 📦 콘텐츠 확장
- **단기(1~2개월):** BGM/SFX 오디오 시스템 통합(참격·피격 음이 수묵 VFX와 프레임 동기), 특수 이벤트 청크 3종(낙하 구간, 가시 벽 이동, 보스 전위 전투) 추가.
- **중기(3~6개월):** 세 번째 캐릭터(먹구름 회피형) 추가 및 3방향 상성 시스템 구현, 일일 도전 모드(지정 캐릭터·제한 조건으로 특정 점수 달성), 수묵 팔레트 스킨 10종 해금 트리.
- **장기(6개월+):** 협동/경쟁 멀티플레이(같은 맵을 두 플레이어가 다른 경로로 동시 진행 후 점수 합산), 커뮤니티 제작 청크 에디터 공개, 전통 회화 작가 협업 한정 아트 팩.

### ⚙️ 기술적 개선
- **ActionFeedback 우선순위 큐:** 같은 프레임에 여러 ActionFeedback 이벤트가 발생할 때 처리 순서를 명시적 우선순위 큐로 관리해 콜아웃·VFX 덮어쓰기 버그 방지.
- **청크 생성 시드 저장:** 현재 무한 청크가 실행마다 랜덤 재생성되는 구조를 시드 기반으로 전환해 특정 명국을 리플레이하거나 공유할 수 있게 함 — 소셜 바이럴 경로 확보.
- **CameraImpactProfile 런타임 튜닝 UI:** 개발 빌드에서 흔들림/줌/플래시 파라미터를 슬라이더로 실시간 조정할 수 있는 디버그 패널을 추가해 QA 시간 단축 및 느낌 조정 반복 비용 절감.
- **Electron 내장 HTTP 서버 포트 충돌 방지:** 현재 고정 포트 사용 시 동일 PC에서 두 인스턴스 실행 시 충돌 가능 — 포트를 동적으로 탐색(`portfinder` 라이브러리)하고 `loadURL`에 런타임 포트를 주입하는 방식으로 전환.
